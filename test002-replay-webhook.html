Test 002: replay-webhook
Objective Ensure it is possible to import an existing git repository.
Test Status Stable
Prerequisite A Postsai installation on a Debian/Ubuntu system.
# Action User input Expected result
1 Checkout replay-webhook git clone https://github.com/postsai/replay-webhook.git Completion without error
2 Change into replay-webhook directory cd replay-webhook Working directoy was changed
3 Display help ./replay.py -h The command usage is displayed
4 Import the repository ./replay.py --home-url=https://github.com/postsai/replay-webhook --url=http://localhost/postsai/api.py --name=replay-webhook . At least line which names the branch "master"
5 Verify Import Open Postsai in you web browser and click on "Search" The Postsai search form displayed.
6 Search for the recently imported repository Repository: replay-webhook
Date: Since the beginning of time
Click on the "Search" button and remember the number of results.		 A number of commits is displayed.
7 Import the repository again ./replay.py --home-url=https://github.com/postsai/replay-webhook --url=http://localhost/postsai/api.py --name=replay-webhook . At least line which names the branch "master"
8 Verify Import Open Postsai in you web browser and click on "Search" The Postsai search form displayed.
9 Search for the recently imported repository Repository: replay-webhook
Date: Since the beginning of time
Click on the "Search" button.		 A number of commits is displayed and the number is equal to step 6.
